# 投放回传数据面板 PRD

版本：V1.2（最终交验版）
关联模块：账户权限、广告对象数据、落地页行为数据、回传策略中心

## 1. 背景与目标

投手需要在同一看板内查看投放、落地页承接、回传策略与异常监控数据，定位异常对象和异常环节；本期不输出自动优化建议，也不在数据面板内修改投放或策略配置。

本期目标：

1. 以平台、账户、广告等筛选范围查看真实数据，支持从账户逐层定位到广告、创意和包名。
2. 提供日趋势、已完成小时趋势及策略组回传结果，辅助定位异常时间与对象。
3. 支持个人监控任务和站内报警；规则命中后能查看报警事实及当前用户的已读状态。

## 2. 页面范围

| 模块 | 页面目标 | 本期主要对象 |
| --- | --- | --- |
| 首页 | 查看账户今日关键指标、近期趋势和未读报警 | 平台、广告账户 |
| 广告对象分析 | 逐层分析账户、广告、创意及 APP包名 | 账户、广告、创意、APP 包名 |
| 落地页分析 | 查看落地页承接漏斗与行为趋势 | 落地页、平台、账户、广告 |
| 策略组执行分析 | 先看策略组总量，再查版本和绑定对象 | 策略组、版本、绑定对象 |
| 监控管理 | 配置个人监控任务并查看报警记录 | 账户、广告、创意、落地页、策略组 |
| 顶部监控报警 | 集中展示当前用户通知并标记已读 | 报警通知 |

不做范围：自动优化建议、外部提醒、共享监控任务、设备系统/手机品牌细分、包名独立监控对象、投放或策略配置编辑、消耗/成本向包名分摊。

## 3. 通用筛选、权限与个人配置

### 3.1 筛选通则

1. 平台为单选；页面仅展示当前用户在该平台有权限的数据。
2. 广告账户为多选且可为空。为空表示当前平台、当前权限下的全部账户，不代表无数据。
3. 广告为多选且可为空。为空表示当前账户范围下的全部广告。
4. 账户、广告等下级候选项随上级筛选范围收敛；变更上级筛选时，已不在范围内的下级选项自动移除。
5. 首页近期趋势支持今天、近 3 天、近 7 天、近 30 天和自定义日期；默认今天并按小时展示。自定义日期最长 30 天；策略组总览使用单个自然日。
6. 分析列表、策略组总览、版本分析和绑定对象分析中，数值指标列均支持升序 / 降序排序；切换排序不改变当前筛选范围和下钻路径。策略组总览、版本分析中同一指标跨多个转化事件时，按该指标的全部事件总和排序。

### 3.2 个人配置

1. 首页关键指标卡不设数量上限，至少保留 1 张；用户可按个人习惯增删。
2. 首页趋势指标、广告对象列表可见列均按用户保存；同一用户再次进入沿用上次配置。
3. 所有保存的个人配置只影响本人，不改变其他用户的页面或数据权限。

## 4. 首页

### 4.1 筛选与指标卡

1. 顶部仅提供平台、广告账户筛选；未选择账户时汇总平台内全部可见账户。
2. 平台、广告账户共同限定首页全部数据范围，包括今日关键指标和近期趋势。
3. 今日关键指标固定按今日、截至当前已完成数据的口径展示，不受近期趋势日期影响。
3. 指标卡候选范围为账户可用指标，包含消耗、成本、曝光、点击、平台转化及真实行为指标。

### 4.2 近期趋势

1. 标题右侧提供时间范围；不在页面顶部筛选区重复展示。
2. 时间范围默认“今天”，按已完成小时展示；支持近 3 天、近 7 天、近 30 天和自定义日期。
3. 默认指标为“消耗 + 转化成本”。
4. 最多同时选择 2 个账户可用指标；使用“趋势指标 1（必选）+ 趋势指标 2（选填）”两个下拉框，不平铺全部指标。每个指标使用独立左右 Y 轴，避免不同量级相互压缩。
5. 时间范围仅影响近期趋势；自定义日期沿用最长 30 天限制。

### 4.3 报警摘要

1. 首页展示当前用户未读报警摘要，包含对象、命中指标/条件、触发时间和查看操作。
2. 顶部“监控报警”和首页摘要同时保留；二者均指向同一条报警通知。
3. 点击查看后，当前用户该条通知实时置为已读；不影响报警事实或其他用户。

## 5. 广告对象分析与 APP包名

### 5.1 对象层级

1. 页面支持“广告账户、广告、创意素材、APP包名”四个视图。
2. 广告对象主路径为：广告账户点击下钻至广告，广告点击下钻至创意。
3. 账户和广告均提供独立“查看趋势”操作；创意仅提供“查看详情”。创意详情已包含趋势与链路数据，不再提供重复的“查看趋势”入口。
4. 页面默认选择“今天”并进入小时模式；小时模式仅支持今天及详情内指定的单个历史自然日。选择近 7 天、近 14 天、近 30 天或指定日期时，自动切换为日模式并按对应日期口径汇总；“自定义日期”需展示指定日期输入项。所有小时趋势默认“今日”，并通过“选择其他日期”指定任意自然日。

### 5.2 列管理与趋势

1. 广告对象列表默认展示全部可用指标；用户可通过“管理列”隐藏或恢复列，配置按个人保存。首页趋势指标下拉同样覆盖广告账户全部可用指标。
2. 广告对象小时趋势可从该对象可用指标中选择，最多同时选择 2 个；默认消耗 + 点击。日期与指标筛选使用紧凑布局；两个下拉框互相排除对方已选项，第二个选填，选择后趋势图即时刷新。
3. 创意详情的趋势与广告对象分析页当前模式保持一致：日模式仅展示链路按日趋势，并继承当前时间范围；小时模式仅展示小时趋势，默认今日，可指定任意历史自然日。两种趋势不在详情内同时展示；详情保留“新建监控”，不再提供重复的“进入小时级分析”操作。

### 5.3 APP包名

1. APP包名是广告对象分析的独立视图，不是创意详情内的临时维度，也不是监控对象类型。
2. 使用平台 → 广告账户 → 广告筛选范围；未选账户或广告时分别表示当前上层范围全量。
3. 支持日汇总和小时趋势；小时趋势默认“今日”，可选择其他任意自然日。
4. 列管理仅展示 APP包名实际可用的行为字段：落地页进入、点击、加粉、开口、线索卡片及适用比例；不展示消耗、任何成本或估算分摊值。
5. 线索卡片数据暂不可用时展示 `-`，不以 0 或估算替代。

## 6. 落地页分析

1. 筛选项为平台、广告账户（多选可空）、广告（多选可空）、时间范围和“筛选落地页”搜索。搜索支持按落地页名称或 ID 模糊匹配，并与其他筛选条件取交集。
2. 平台、账户、广告、日期共同限定列表、详情漏斗和详情小时趋势的数据聚合范围。
3. 广告筛选只用于收敛数据范围，不提供广告来源归因、反向下钻或创意归因。
4. 落地页列表展示进入、点击、加粉、开口、线索卡片的次数/比例；不展示消耗或成本。
5. 点击落地页进入详情，展示“进入 → 点击 → 加粉 → 开口 → 线索卡片”漏斗、阶段转化率和小时趋势。
6. 小时趋势默认“今日”，可选择其他任意自然日；可选择全部落地页可用行为指标，最多同时选择 2 个，默认落地页进入次数 + 落地页点击次数。日期与指标筛选使用紧凑布局；两个下拉框互相排除对方已选项，第二个选填，选择后即时刷新。
7. 详情标题右侧提供“新建监控”，自动预填当前落地页；详情内不提供重复的小时趋势跳转按钮。

## 7. 策略组执行分析

### 7.1 策略组总览

1. 策略组页顶部提供平台、广告账户和数据日期筛选；默认“今日”，选择其他日期后可指定任一自然日。筛选范围共同限定总览、策略组候选项、版本分析和绑定对象分析数据。
2. 使用“总览 / 版本分析 / 绑定对象分析”页签切换查看维度，不通过点击策略组卡片跳转。
3. 总览默认展示当前筛选范围内所有可见策略组；直接进入版本分析或绑定对象分析时不预选策略组，运营需手动选择。仅从总览内某策略组的入口跳转时，自动带入该策略组。
4. 策略组候选项仅展示当前平台、广告账户范围内可见的策略组，并支持按策略组名称模糊搜索。
5. 每个策略组按四个转化事件展示：落地页点击、加粉、开口、生成线索卡片。
6. 总览使用紧凑表格展示：策略组作为左侧合并表头，转化事件分行；“实际产生、回传成功、扣量、回传失败”为统一指标列，并复用版本表的浅色事件底纹、版本/组间加粗分隔线和指标说明入口。
7. 每个策略组的合并表头内提供“查看版本”“查看绑定对象”入口；点击后自动选中该策略组，并切换至对应页签。

### 7.2 版本与绑定对象

1. 版本详情使用紧凑表格展示，策略版本作为左侧合并表头；同一版本下按转化事件分行，统一展示实际产生、回传成功、扣量、回传失败，避免重复卡片标题与多余行高。相同事件使用浅色底纹区分，版本之间使用加粗分隔线。
2. 绑定对象作为独立页签展示；同一对象在当前筛选日期内关联多个版本时，按“对象 + 版本”分别成行，并提供“回传总量 / 所处状态”二级切换。
3. “回传总量”按当前平台、账户和日期范围汇总，展示对象名称/ID、策略组、对象类型、策略版本，以及落地页点击、加粉、开口、生成线索卡片四个事件；每个事件同时展示实际、成功、扣量、失败四类数量。点击事件列表头时，按该事件的“实际产生”数切换升序 / 降序；在页签旁或表格右上角展示该排序口径说明。
4. 回传口径必须满足：`实际产生数量 = 回传成功量 + 扣量数量 + 回传失败量`；四类回传指标标题旁提供可点击的 `i`，在表头下方气泡完整展示指标统计口径。
5. 回传失败量为除成功回传、策略扣量外的未回传量；本期不在总览拆分失败原因。

### 7.3 小时趋势与状态

1. 从绑定对象行点击“查看趋势”后，以右侧抽屉打开小时趋势；默认“今日”，可选择其他任意自然日。通过单个“转化事件”下拉框选择 1 个事件，再展示该事件的实际产生、回传成功、扣量、回传失败四条线。
2. 不将不同转化事件混在同一张小时趋势图中。
3. 绑定对象“所处状态”展示当前实时状态，不随数据日期变化；字段包括当前阶段、阶段进度、命中分支、脱离条件摘要和阶段停留时长。仅对象当前仍在该策略版本下时展示前四项实时状态；历史版本不展示当前阶段、进度、分支和脱离条件，统一显示 `—`，但保留离开该版本前最后所在阶段的停留时长。
4. 绑定对象运行状态仅用于查看当前状态及进入对应运行详情，不提供状态趋势图。

## 8. 监控任务

### 8.1 对象模型

| 对象类型 | 选择链路 | 全选范围 |
| --- | --- | --- |
| 广告账户 | 平台 → 广告账户 | 当前平台、当前权限下全部可见账户 |
| 广告 | 平台 → 广告账户 → 广告 | 当前账户范围内全部可见广告 |
| 创意素材 | 平台 → 广告账户 → 广告 → 创意 | 当前账户/广告范围内全部可见创意 |
| 落地页 | 平台 → 落地页 | 当前平台、当前权限下全部可见落地页 |
| 策略组 | 直接选择策略组 | 当前权限下全部可见策略组 |

1. 广告、创意素材按“广告账户 → 广告 → 最终对象”级联选择；广告账户、广告、创意等每一级候选值均支持名称搜索。落地页无需选择广告账户或广告范围，直接在当前平台内选择最终对象；广告账户类型直接在当前平台内选择最终对象；策略组直接选择最终对象。
2. 最终对象使用紧凑的可搜索多选下拉选择器，不在页面平铺全部枚举值；提供“全选当前范围”“清空”操作。
3. 全选时将当前权限和上级筛选范围内的实际对象 ID 写入规则；保存后对象集合固定，不自动纳入后续新增对象。
4. 上级筛选变化导致已选下级对象越界时，自动从待保存对象集合移除。
5. 规则保存后若对象失权、删除或不可用，系统从有效监控对象中排除并保留规则说明；剩余对象继续生效。

### 8.2 指标与条件

1. 监控指标复用对应分析列表的全部可用指标，不单独维护一份缩减字段。
2. 落地页和包名不可使用消耗、成本类指标；包名不作为独立监控对象。
3. 支持多个指标条件；每个条件包含指标、比较方式（大于、大于等于、等于、小于等于、小于）和阈值。比较方式统一使用中文文案；阈值仅允许大于等于 0 的数字，最多保留 3 位小数。指标为比例时，在输入框后展示 `%`，填写值按百分数口径理解。
4. 任一有效对象命中任一有效条件即触发报警；报警记录精确记录实际命中的一个对象和一个条件。

### 8.3 时间、频率与可见性

1. “近 X 小时”为选填，填写范围为 1–24 的整数；统计最近 N 个已完成自然小时，不包含当前未完成小时。
2. 未填写近 X 小时时，统计当日 00:00 至最近已完成自然小时的累计值。
3. 检测频率固定为 5 分钟、15 分钟、30 分钟、1 小时；保留生效时段。
4. 监控名称置于新建任务表单顶部，可选填。未填写时自动命名为“对象类型 - 首条条件指标 - 创建日期”，并在输入框下实时展示当前将生成的名称。
5. 任务默认仅创建人可见、编辑和接收报警；本期不提供共享范围或同数据范围共享。
6. 同一用户不能保存完全相同的规则；对象类型、对象集合、指标条件集合、时间窗口、检测频率和生效时段相同即视为重复，集合比较忽略排列顺序。

## 9. 顶部报警与报警记录

1. 顶部“监控报警”展示当前用户可接收的通知，未读优先、按触发时间倒序。
2. 点击顶部通知、首页摘要查看或报警记录详情时，当前用户该条通知置为已读，并实时刷新未读数；已读不代表异常恢复。
3. 监控管理保留“报警记录”页签，展示报警 ID、触发时间、通知状态、对象、命中指标、命中条件、统计范围和详情操作。
4. 报警记录保留历史触发事实，但本期不提供“查看小时趋势”入口。
5. 不展示“已处理”“已恢复”等处置状态。

## 10. 数据、权限与异常规则

1. 过滤器、列表、趋势、对象选择器、监控任务与报警通知必须使用同一份数据权限结果。
2. 无权限、无数据或指标不可用时展示明确空状态或 `-`，不以 0、历史值或估算补齐。
3. 当前未完成小时展示“数据计算中”，不得使用不完整小时数据触发监控。
4. 今天展示已完成小时；历史自然日展示完整小时。策略组总览的“今天”口径为截至当前可用数据的当日累计。
5. Demo 可不单独提供“数据计算中 / 无权限 / 无数据”状态切换入口；但真实实现必须按本节规则返回明确状态，不得以 0、历史值或估算值替代。

## 11. 验收清单

1. 账户、广告不选时，首页、广告对象、落地页和 APP包名均按当前权限范围全量汇总。
2. 首页、广告对象、落地页趋势最多选择 2 个指标，且用户配置刷新后保留。
3. APP包名可在账户范围、广告范围下查看日/小时数据，且无消耗或成本字段。
4. 策略组卡片、版本卡片及绑定对象记录与回传闭环公式一致。
5. 广告账户、创意的“全选当前范围 / 清空”可用；全选保存后为固定对象集合。
6. 近 X 小时留空和填写两种监控口径均只使用已完成小时；任一条件命中能生成报警。
7. 顶部通知与报警记录详情打开后同步已读；报警记录页无小时趋势入口。
